# RACEF 框架 (RACEF Framework)

> 来源：[Juuzt AI - RACEF Framework](https://juuzt.ai/knowledge-base/prompt-frameworks/the-racef-framework/)

## 框架概述

RACEF 框架（Rephrase, Append, Contextualize, Examples, Follow-Up）是 AI 提示工程中的前沿方法论。它通过结构化、动态的方式来制作精确的提示，帮助用户与大语言模型（LLM）进行高效交互，获得详细、相关且富有创造性的输出。

---

## 五大核心组件

| 组件 | 英文 | 说明 |
|------|------|------|
| **重述** | Rephrase | 优化查询的措辞，以获得更精确和相关的结果 |
| **追加** | Append | 添加细节或约束条件，引导 AI 产出目标化的输出 |
| **上下文化** | Contextualize | 提供背景信息或情境细节，使 AI 的响应与你的需求对齐 |
| **示例** | Examples | 包含样本输入或期望输出，以明确期望 |
| **跟进** | Follow-Up | 通过允许 AI 提问或完善响应来鼓励迭代改进 |

---

## RACEF vs CRISPE 对比

| 方面 | RACEF | CRISPE |
|------|-------|--------|
| **聚焦点** | 优化提示的精确性和适应性 | 追求清晰度和迭代优化以获得一致结果 |
| **最适合** | 迭代性和复杂问题解决 | 结构化和分步骤任务 |
| **灵活性** | 跨领域高度灵活 | 专注于结构化用例 |

**结论**：RACEF 适合动态、探索性任务，而 CRISPE 适合需要一致输出的场景。

## 提示示例

### 示例1：市场研究报告

**场景**：为科技初创公司创建市场研究报告

```
Rephrase（重述）: 分析可穿戴技术的最新趋势。
Append（追加）: 包括关键增长因素和消费者偏好。
Contextualize（上下文化）: 聚焦北美市场，时间跨度为过去五年。
Examples（示例）: 提供该领域三个成功产品的案例。
Follow-Up（跟进）: 建议进入该市场的可执行洞察。
```

### 示例2：教育课程开发

**场景**：设计可再生能源课程

```
Rephrase（重述）: 创建可再生能源概念的入门指南。
Append（追加）: 增加对太阳能和风能应用的关注。
Contextualize（上下文化）: 为高中生量身定制内容。
Examples（示例）: 包含城市地区可再生能源项目的案例研究。
Follow-Up（跟进）: 建议用于实践学习的互动活动。
```

### 示例3：客户保留策略

**场景**：制定提高客户保留率的策略

```
Rephrase（重述）: 识别订阅模式中客户流失的关键原因。
Append（追加）: 包括客户行为趋势的数据分析。
Contextualize（上下文化）: 聚焦面向小企业的 SaaS 公司。
Examples（示例）: 提供三个已被证明有效的保留策略。
Follow-Up（跟进）: 建议对当前保留方法的改进。
```

### 示例4：财务预测

**场景**：为零售连锁店创建财务预测

```
Rephrase（重述）: 预测下一财年的收入增长。
Append（追加）: 包括市场趋势和季节性模式等变量。
Contextualize（上下文化）: 聚焦城市地区的服装行业。
Examples（示例）: 使用类似零售连锁店的历史数据。
Follow-Up（跟进）: 建议减轻表现不佳地区风险的策略。
```

### 示例5：数字营销策略

**场景**：为金融科技应用创建数字营销策略

```
Rephrase（重述）: 设计一个增加应用下载量的营销活动。
Append（追加）: 聚焦社交媒体和网红合作。
Contextualize（上下文化）: 目标为城市地区的千禧一代和Z世代。
Examples（示例）: 突出成功的金融科技营销活动案例。
Follow-Up（跟进）: 推荐追踪活动成功的指标。
```

### 示例6：企业培训项目

**场景**：为远程团队设计企业培训项目

```
Rephrase（重述）: 开发一个改善远程团队协作的培训项目。
Append（追加）: 包括沟通工具和团建活动模块。
Contextualize（上下文化）: 为拥有全球团队的科技公司量身定制。
Examples（示例）: 突出成功的远程团队培训项目案例研究。
Follow-Up（跟进）: 建议评估项目效果的指标。
```

### 示例7：网络安全风险评估

**场景**：为金融公司进行网络安全风险评估

```
Rephrase（重述）: 识别组织网络中的潜在网络安全风险。
Append（追加）: 包括钓鱼和数据泄露等风险类别。
Contextualize（上下文化）: 聚焦中型金融机构。
Examples（示例）: 参考近期网络安全漏洞案例研究。
Follow-Up（跟进）: 提出可执行的步骤来降低已识别的风险。
```

### 示例8：员工健康项目

**场景**：为员工设计健康项目

```
Rephrase（重述）: 开发一个改善心理健康的员工健康项目。
Append（追加）: 包括工作坊、一对一辅导和正念课程。
Contextualize（上下文化）: 聚焦高压力行业的公司。
Examples（示例）: 参考科技和金融行业的成功项目。
Follow-Up（跟进）: 建议评估项目成功的指标。
```

## 最佳配合的生成式 AI 模型

| 模型 | 优势 |
|------|------|
| **OpenAI GPT-4** | 非常适合迭代问题解决和细致入微的响应，利用其深度上下文理解能力 |
| **Google Bard/Gemini** | 适合创意头脑风暴和需要多元视角的上下文驱动任务 |
| **Anthropic Claude** | 擅长为复杂的多步骤查询优化和上下文化响应 |
| **Meta Llama 2** | 适合有技术或学术焦点的特定领域任务 |

## 框架优缺点

### ✅ 优势

- **灵活性**：适应各种任务，从创意内容到技术分析
- **结构化**：提供清晰的分步方法来制作有效提示
- **迭代优化**：鼓励跟进和澄清以获得更深入的洞察

### ⚠️ 劣势

- **学习曲线**：需要熟悉提示工程概念才能最佳使用
- **耗时**：为简单任务制作包含所有元素的详细提示可能比较费时

- 📈 财务预测和分析

## 总结

RACEF 框架是任何希望掌握 AI 提示工程的人的强大工具。通过结合结构与灵活性，它使用户能够创建产出精确、相关且富有创造性输出的提示。无论你是在制定营销策略、分析数据还是开发教育内容，RACEF 都能提供最大化 LLM 潜力所需的清晰度和深度。
